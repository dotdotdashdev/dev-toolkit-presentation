The Problem

Every AI session starts from zero.


  • No memory of your team's naming conventions or standards
  • No knowledge of your project's history, goals, or constraints
  • No shared context across the team
  • Everyone builds their own prompts, none of it shared
This is the core problem. Every new session is a blank slate. Devs re-explain branch conventions. Designers re-explain creative tenets. Producers re-explain project timelines. At team scale it gets worse. Everyone has their own prompt snippets, their own workarounds, none of it version controlled or shared.

What is the Dot Toolkit?

A shared repo of skills and context files
that any AI coding tool can read.


So the assistant already knows how your team works.

One sentence pitch. It's a shared repo. Skills and context files. Any AI tool reads them. Your assistant starts the session already knowing your conventions.

Shared Context: What We're Exploring

The bigger opportunity: sharing project context across teams and their agents.

What to share

Project background & goals
Plans, tasks, and statuses
Conventions & standards

How we're exploring it

MCP server (structured to-dos, human-readable)
Plans & tasks stored in GitHub
SurrealDB → flat files in repo
3rd party context & orchestration tools


All approaches keep content in GitHub with need-to-know data only in MCP, designed to limit client data exposure.

This is where it gets interesting beyond skills. We want agents and people to share context: project background, task statuses, who's blocked on what. We're exploring multiple methods: an MCP server that can serve structured to-dos and is human-readable, storing plans and to-dos in GitHub, using SurrealDB to render data into flat files shared in repo, and evaluating 3rd party tooling for context management and agent orchestration. All of these keep content in GitHub by default. Only need-to-know data lives in MCP. Ideally this would be a company-approved cloud system, but we've been blocked on that, so we're working within constraints.

Where This Is Going

This is a flag in the ground on the path to orchestrating automated agents.

  • Today: shared skills and context that make AI assistants meaningfully better across roles
  • Next: shared project context and task statuses, enabling collaboration between teams and their agents
  • Ahead: orchestrated agents that can coordinate work, hand off tasks, and keep everyone in sync
  • Certain: this problem isn't going away. We're learning by building
The skills repo is step one. Shared context is step two. The endgame is orchestrated agents that can work across teams. Not replacing people, but coordinating work. We're building toward that incrementally. Each step delivers value on its own.
