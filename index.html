The Problem

Every AI session starts from zero.


  • No memory of your branch naming conventions
  • No knowledge of your deploy targets
  • No shared context across the team
  • Everyone builds their own prompts — none of it shared
This is the core problem. Every new session is a blank slate. At team scale it gets worse — everyone has their own prompt snippets, their own workarounds, none of it version controlled or shared.

What is dev-toolkit?

A shared repo of skills and context files
that any AI coding tool can read.


So the assistant already knows how your team works.

One sentence pitch. It's a shared repo. Skills and context files. Any AI tool reads them. Your assistant starts the session already knowing your conventions.

This is the core mental model. Skills tell the AI WHAT to do. Context tells it HOW to behave. Both are just markdown files in a repo.

Where We're Headed

This is a flag in the ground.

  • Works today: skills and context that make AI assistants meaningfully better at day-to-day work
  • Theoretical: shared context across agents and people — the MCP server exists but isn't battle-tested
  • Emerging: entire.io, Temporal, and other tools that could reshape team-level AI context
  • Certain: this problem isn't going away — we're learning by building
Be honest. We're weeks into this, not months. The shared context server is theoretical — hasn't been tested with multiple people. Entire.io just launched and could be a big player. Temporal is on the radar but unexplored. What we DO have is a working set of skills that deliver value right now. The point: we started, and we're iterating.
